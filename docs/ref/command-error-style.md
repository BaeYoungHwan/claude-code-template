# 커맨드 오류 출력 스타일 가이드

> 모든 `.claude/commands/` 스킬이 사용하는 출력 기호 표준입니다.

---

## 출력 기호 표준

| 기호 | 의미 | 사용 조건 |
|------|------|---------|
| `✅` | 성공 | 단계 완료, 검사 통과 |
| `ℹ️` | 정보 | 현황 안내, 사용자 액션 필요 없음 |
| `⚠️` | 경고 | 계속 진행하지만 주의 필요 |
| `⏭️` | 건너뜀 | 조건 미충족으로 단계 스킵 |
| `⛔` | 차단 | 사용자 작업 요건 미충족으로 즉시 중단 |
| `❌` | 실패 | 오류 발생, 종료 |
| `🔴` | 보안/심각 | 즉시 수정 필요한 보안·데이터 위험 |
| `🟡` | 경고 (보안) | 권장 수정 사항 |
| `🟢` | 통과 (보안) | 검토 완료, 이상 없음 |

---

## 건너뜀 형식

단계를 건너뛸 때는 반드시 아래 형식으로 표시합니다:

```
⏭️  [N단계] 건너뜀 — [이유]
```

예시:
```
⏭️  [2단계] 건너뜀 — score.py 없음
⏭️  [8단계] 건너뜀 — .env에 SMTP 설정 없음
```

---

## 차단 형식

사용자가 진행할 수 없는 조건을 감지했을 때:

```
⛔ [사유]
   [해결 방법]
```

예시:
```
⛔ main/master 브랜치에서는 PR을 생성할 수 없습니다.
   작업 브랜치로 전환 후 다시 시도하세요:
     git checkout -b feature/[작업명]
```
