# 품질 문서 가이드

> 이 가이드는 QUALITY_SCORE.md · RELIABILITY.md · tech-debt-tracker.md 의 목적과 작성 방법을 설명합니다.

---

## QUALITY_SCORE.md

- **목적**: 각 Phase 완료 후 코드 품질을 A~F 등급으로 기록
- **작성 시점**: Phase 완료 직후 (테스트 커버리지 측정 후)
- **작성자**: 개발자 직접 입력
- **방법**: 테스트 커버리지 % 기반으로 등급 결정 후 해당 Phase 행에 기입

## RELIABILITY.md

- **목적**: 배포 후 런타임 안정성 지표 기록 (업타임, 에러율, p99 레이턴시)
- **작성 시점**: 배포 후 운영 중
- **작성자**: 개발자 직접 입력
- **방법**: 모니터링 도구에서 수치 확인 후 기입. 미배포 프로젝트는 비워도 됨

## tech-debt-tracker.md

- **목적**: 코드베이스의 기술 부채 자동 추적
- **작성 시점**: doc-gardener 에이전트 실행 시 자동 기록
- **작성자**: doc-gardener 에이전트 자동 추가 (수동 추가도 가능)
- **방법**: doc-gardener가 코드-문서 불일치 발견 시 `doc-drift` 유형으로 자동 기입

---

## 파일 생성 방법

`QUALITY_SCORE.md`와 `RELIABILITY.md`는 **Phase 첫 완료 시점**에 프로젝트 루트에 생성합니다.
미리 생성할 필요 없으며, `/init-project` 시에는 생성하지 않습니다.

### QUALITY_SCORE.md 템플릿

```markdown
<!-- 작성 방법: docs/ref/quality-guide.md 참고 -->

# 품질 점수

| Phase | 완료일 | 테스트 커버리지 | 등급 | 비고 |
|-------|--------|----------------|------|------|
| Phase 1 | - | - | - | |

## 등급 기준

| 등급 | 커버리지 |
|------|---------|
| A | 90% 이상 |
| B | 75~89% |
| C | 60~74% |
| D | 45~59% |
| F | 44% 이하 |
```
