# /init-project — 새 프로젝트 초기화

새 프로젝트를 시작할 때 실행하세요. Claude가 양식을 출력하면, 직접 작성해서 붙여넣으면 됩니다.

## 실행 절차

**Step 1 — 아래 양식을 사용자에게 출력한다:**

```
=== 프로젝트 초기화 양식 ===
(항목을 작성 후 전체를 붙여넣어 주세요)

1. 프로젝트명:
   예) my-saas-app

2. 기술 스택:
   예) Python 3.12 + FastAPI + PostgreSQL + React 18

3. MVP 핵심 기능 (반드시 구현할 것):
   예)
   - 사용자 로그인/회원가입
   - 대시보드 조회
   - 데이터 CSV 업로드

4. MVP 제외 사항 (이번 버전에 포함하지 않을 것):
   ※ AI가 알아서 추가 구현하는 것을 막는 범위 선언입니다.
   예)
   - 소셜 로그인 (Google, GitHub)
   - 이메일 알림 기능
   - 다국어(i18n) 지원

5. 제약사항 / 특이사항:
   예) 클라이언트 측 렌더링만 허용, 외부 API 의존 최소화

==============================
```

**Step 2 — 사용자가 양식을 작성해서 제출하면:**

1. 내용을 검토하고 불명확한 부분에 대해 **2~3가지 보완 질문**을 한다
   - MVP 제외 사항이 비어있으면 반드시 물어볼 것
   - 스택이 모호하면 버전이나 DB 선택 이유를 물어볼 것

2. 보완 답변을 받으면:
   - `CLAUDE.md` 상단 플레이스홀더(`[프로젝트명]`, `[프로젝트 한 줄 설명]`)를 채운다
   - `docs/product-specs/PRD-v1.md`를 생성한다 (아래 PRD 초안 구조 사용)

**PRD 초안 구조:**

```markdown
# [프로젝트명] PRD v1

## 개요
[프로젝트 한 줄 설명]

## 기술 스택
[스택]

## MVP 핵심 기능
[기능 목록]

## MVP 제외 사항
[제외 목록]
※ 위 항목은 Claude가 임의로 구현하지 않습니다.

## 제약사항
[제약사항]

## 다음 단계
- [ ] 아키텍처 설계 (`docs/design-docs/architecture-layers.md` 참조)
- [ ] Phase 분할 후 `docs/exec-plans/active/`에 실행 계획 생성
```
